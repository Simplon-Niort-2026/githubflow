Pourquoi ne travailles-tu pas uniquement sur main ?
Pour ne pas faire des conflits avec le travail des autres sur un même projet.

Pourquoi éviter de créer une seule branche par personne ?
Éviter une branche par personne permet d’éviter l’isolement du code, les divergences massives et la perte de contrôle sur la qualité des contributions.

Qu'est-ce qu'un workflow ?
Un workflow (ou flux de travail) représente la suite d’étapes, humaines ou automatisées, qui permettent de réaliser un objectif : qui fait quoi, dans quel ordre, avec quelles informations et quelles validations.
Il décrit comment un travail avance d’une étape à l’autre, jusqu’à son achèvement.

Quand créer une branche avec le GitHub Flow ?
Après avoir créé une issue, on créé une nouvelle branche à partir de cette issue.

Qu'est-ce qu'une Pull Request ?
C'est la dernière étape avant de pousser son code sur une autre branche, elle va permettre de faire vérifier son code par un reviewer et aussi gérer les éventuelles conflits 

Qu'est-ce qu'une Code Review ?
C'est le fait de faire une relecture d'un code d'une pull request, de vérifier qu'il n'y à pas d'incohérences, que celui ci est propre et fonctionnel. Ainsi on évite au maximum les érreurs.
