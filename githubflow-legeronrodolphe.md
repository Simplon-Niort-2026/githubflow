# Pourquoi ne travailles-tu pas uniquement sur `main` ?
La branche principale `main` est réservée pour le code principal.
La modification du code ou l'ajout de fonctionnalité sur d'autres branches permet d'avoir une branche principale toujours propre, sans bug.
la branche `main` est stable et déployable.
# Pourquoi éviter de créer une seule branche par personne ?
La bonne pratique est de créer une branche par `feature` (fonctionnalité)
# Qu'est-ce qu'un workflow ?
Le workflow signifie "méthode de travail".
C'est la procédure à respecter pour la bonne entente et réalisation au sein d'une équipe.
# Quand créer une branche avec le GitHub Flow ?
Une branche est créée à partir de `main` pour toute modification, quelle que soit son ampleur.

# Qu'est-ce qu'une _Pull Request_ ?
Les pull request sont utilisées pour initier des discussions (code review) sur une branche avant de la fusionner dans `main`
# Qu'est-ce qu'une _Code Review_ ?
La revue de code  est l'examen systématique du code source par une tierce personne. Son objectif est d'identifier les erreurs et mauvaises pratiques avant la mise en production, mais aussi d'améliorer la qualité et la sécurité du code.
